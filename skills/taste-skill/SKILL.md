# tasteskill: Anti-Slop Frontend Skill
> Landing pages, portfolios, and redesigns. Not dashboards, not data tables, not multi-step product UI.
> Every rule below is contextual. None of it fires automatically. First read the brief, then pull only what fits.

## 0. BRIEF INFERENCE (Read the Room Before Anything Else)
Before touching code or tweaking dials, infer what the user actually wants.
0.A Read these signals: page kind; vibe words; reference signals (URLs/screenshots/brands); audience; existing brand assets; quiet constraints (a11y, public-sector, regulated, kids, trust-commerce — these OVERRIDE aesthetic).
0.B Output a one-line Design Read before generating: "Reading this as: <page kind> for <audience>, with a <vibe> language, leaning toward <design system or aesthetic family>."
0.C If ambiguous, ask exactly ONE clarifying question. If confident, don't ask — declare and proceed.
0.D Anti-Default Discipline: do NOT default to AI-purple gradients, centered hero over dark mesh, three equal feature cards, glassmorphism on everything, infinite micro-animations everywhere, Inter + slate-900.

## 1. THE THREE DIALS
- DESIGN_VARIANCE: 8 (1=Perfect Symmetry, 10=Artsy Chaos)
- MOTION_INTENSITY: 6 (1=Static, 10=Cinematic/Physics)
- VISUAL_DENSITY: 4 (1=Art Gallery/Airy, 10=Cockpit/Packed Data)
Baseline 8/6/4 unless the design read overrides. Overrides happen conversationally, not by editing this file.
1.A Dial inference: minimalist/clean/Linear → V5-6 M3-4 D2-3; premium consumer/Apple → V7-8 M5-7 D3-4; playful/Awwwards/agency → V9-10 M8-10 D3-4; landing/portfolio default → V7-9 M6-8 D3-5; trust-first/public-sector → V3-4 M2-3 D4-5; redesign-preserve → match/+1/match; redesign-overhaul → +2/+2/match.
1.C Use these exact variable names; never invent aliases.

## 2. BRIEF → DESIGN SYSTEM MAP
2.A Reach for official packages when the brief matches: Microsoft/enterprise → Fluent UI; Google/Material → @material/web; IBM analytics → Carbon; Shopify → Polaris; Atlassian → Atlaskit; GitHub devtool → Primer; UK public → govuk-frontend; US public → uswds; fast agency MVP → Bootstrap 5.3; accessible React → @radix-ui/themes; owned SaaS components → shadcn/ui (never default state); Tailwind SaaS/AI marketing → Tailwind v4. Honesty rule: use the OFFICIAL package, don't hand-recreate its CSS, don't import tokens then override 90%. One system per project.
2.B Aesthetics with no official package (build native CSS + Tailwind, label borrowed vs official): glassmorphism, bento, brutalism, editorial, dark tech, aurora/mesh, kinetic type, Apple Liquid Glass (no official liquid-glass.css — approximate + label).

## 3. DEFAULT ARCHITECTURE
3.A Stack: React/Next (RSC default; motion/scroll/pointer components are isolated 'use client' leaves). Tailwind v4 (@tailwindcss/postcss or Vite plugin). Animation: Motion (import from motion/react). Fonts: next/font or self-host @font-face + font-display:swap; never <link> Google Fonts in production.
3.B State: local useState/useReducer; global only to avoid deep drilling (Zustand/Jotai/context). NEVER useState for continuous input (mouse/scroll/pointer) — use useMotionValue/useTransform/useScroll.
3.C Icons: Phosphor / Hugeicons / Radix / Tabler (priority). Lucide discouraged. NEVER hand-roll SVG icons. One family per project. Standardize strokeWidth.
3.D Emoji: discouraged by default; allow only for explicit playful/chat/social vibe, sparingly.
3.E Responsive: standard breakpoints; contain with max-w-[1400px]/max-w-7xl mx-auto; min-h-[100dvh] not h-screen for heroes; CSS Grid over flex percentage math.
3.F Verify deps in package.json before importing; output install command if missing.

## 4. DESIGN ENGINEERING DIRECTIVES
4.1 Typography: display text-4xl md:text-6xl tracking-tighter leading-none; body text-base leading-relaxed max-w-[65ch]. Inter discouraged as default (Geist/Outfit/Cabinet Grotesk/Satoshi first; Inter OK for neutral/Linear/a11y). SERIF VERY DISCOURAGED as default — "creative brief = serif" is the #1 AI tell. Serif only if brand names one OR genuinely editorial/luxury/heritage with articulated reason. BANNED serif defaults: Fraunces, Instrument_Serif. Emphasis within a headline = italic/bold of the SAME font, never a random serif word in a sans headline. Italic descender clearance: display italic words with y g j p q need leading-[1.1] min + pb-1/mb-1.
4.2 Color: max 1 accent, saturation <80%. THE LILA RULE: AI purple/blue glow discouraged; neutral bases (zinc/slate/stone) + one high-contrast accent. Override only if brand asks for purple. One palette per project. COLOR CONSISTENCY LOCK: one accent across the whole page. PREMIUM-CONSUMER PALETTE BAN: no default beige/cream + brass/clay/oxblood/ochre + espresso. Rotate alt families (cold luxury, forest, black+tan, cobalt+cream, terracotta+slate, olive+brick, monochrome+one pop); don't repeat the same family twice in a row.
4.3 Layout: ANTI-CENTER BIAS when VARIANCE>4 (split-screen, left-content/right-asset, asymmetric whitespace, scroll-pinned). Center OK for editorial/manifesto/launch.
4.4 Materiality: cards only when elevation = real hierarchy (else border-t/divide-y/whitespace). Tint shadows to bg hue, no pure-black shadows on light. VISUAL_DENSITY>7 → no card boxes. SHAPE CONSISTENCY LOCK: one corner-radius scale, documented rule if mixed.
4.5 Interactive states: build loading (skeletons not spinners), empty, error (inline for forms), tactile :active (-translate-y-[1px]/scale-[0.98]). BUTTON CONTRAST CHECK (WCAG AA 4.5:1 body, 3:1 large; no white-on-white; scrim ghost buttons over photos). CTA WRAP BAN (one line desktop, ≤3 words primary). NO DUPLICATE CTA INTENT (one label per intent across nav/hero/footer). FORM CONTRAST CHECK.
4.6 Forms: label ABOVE input, helper present in markup, error BELOW, gap-2. No placeholder-as-label ever.
4.7 Layout Discipline (hard rules): Hero fits initial viewport (headline ≤2 lines, subtext ≤20 words AND ≤3-4 lines, CTA visible). Plan font + image size together (default text-4xl md:5xl lg:6xl; 6xl/7xl only for 3-5 word headlines). HERO TOP PADDING max pt-24. HERO STACK max 4 text elements (eyebrow OR brand strip; headline; subtext; 1 primary + max 1 secondary CTA) — no tagline below CTAs, no trust micro-strip, no pricing teaser, no bullets, no avatar row in hero. "Used by" wall goes UNDER hero. Nav ONE line desktop, height ≤80px (default 64-72). Bento needs rhythm; EXACT cell count (N items → N cells, no empty tiles). Section-Layout-Repetition Ban (each layout family max once; ≥4 families across 8 sections). ZIGZAG cap: max 2 consecutive image+text splits. EYEBROW RESTRAINT: max 1 eyebrow per 3 sections (hero counts as 1); count `uppercase tracking` labels — fail if > ceil(sections/3); prefer dropping the eyebrow. SPLIT-HEADER BAN (stack headline+body vertically, max-w-65ch). Bento Background Diversity (2-3 cells with real image/gradient/pattern). Mobile collapse explicit per section.
4.8 Images: gen-tool first; else real photos (picsum.photos/seed/{desc}/{w}/{h}); last resort labeled placeholder slots + tell the user. Even minimalist sites need 2-3 real images. Real SVG logos for social proof (Simple Icons/devicon or generated monogram), light+dark, LOGO-ONLY (no category labels). Hand-rolled decorative SVG strongly discouraged. Div-based fake screenshots BANNED. Hero needs a real visual.
4.9 Content density: per section ≤8-word headline + ≤25-word sub + one asset/CTA. No data-dump sections; long lists (>5) use a real component (2-col split, card grid, tabs/accordion, scroll-snap pills, carousel, marquee) not `<ul> divide-y`. Spec sheets: card grid / grouped chunks / featured-vs-rest, not a hairline under every row. COPY SELF-AUDIT: re-read every visible string; kill broken grammar, unclear referents, AI wordplay, mock-poetic meta. Fake-precise numbers flagged unless real or labeled mock. One copy register per page.
4.10 Quotes: max 3 lines body, no em-dash, attribution = name + role (+company), real typographic quotes or none.
4.11 PAGE THEME LOCK: one theme (light/dark/auto) whole page; sections don't invert. Section-level tints within the same family are fine. Set theme once at root.

## 5. CONTEXT-AWARE PROACTIVITY (tools, not defaults)
Glass/glassmorphism (premium/Apple/luxury/media only; add inner border + inset highlight; reduced-transparency fallback). Magnetic physics when MOTION>5 + premium/playful (motion values, never useState). Perpetual micro-interactions when MOTION>5 and the section benefits; spring physics not linear. "Motion claimed = motion shown": if MOTION>4 the page must actually move (hero entry, scroll-reveal, CTA hover); else drop to 3 and ship clean static. MOTION MUST BE MOTIVATED (hierarchy/storytelling/feedback/state; if you can't say why in one sentence, drop it). MARQUEE max one per page. Sticky-stack and horizontal-pan use start:"top top", pin:true, correct scrub (canonical GSAP skeletons in 5.A/5.B; prefer Motion whileInView for simple reveals — 5.C).
5.D Forbidden animation patterns: window.addEventListener("scroll") BANNED; no scrollY in React state; no rAF loops touching React state; use useScroll/ScrollTrigger/IntersectionObserver/CSS animation-timeline.

## 6. PERFORMANCE & A11Y
6.A Animate only transform/opacity; will-change sparingly. 6.B prefers-reduced-motion mandatory above MOTION>3 (infinite loops/parallax/scroll-hijack/magnetic collapse to static). 6.C Dual dark mode by default; Tailwind dark: or CSS vars, one strategy; WCAG AA (AAA body); respect prefers-color-scheme. 6.D CWV: LCP<2.5s (hero priority/preload), INP<200ms, CLS<0.1. 6.E Grain/noise only on fixed pointer-events-none pseudo-elements, never scrolling containers; lazy-load below-fold heavy libs. 6.F Z-index restraint (systemic layers only).

## 7. DIAL DEFINITIONS
VARIANCE 1-3 symmetric grid; 4-7 offsets/overlaps/varied ratios; 8-10 masonry/fractional/huge empty zones (mobile <768px collapses to single column). MOTION 1-3 hover/active only; 4-7 fluid CSS transitions + delay cascades; 8-10 scroll-triggered/parallax/scroll-driven (never addEventListener scroll). DENSITY 1-3 py-32..48; 4-7 py-16..24; 8-10 tight, 1px lines not cards, font-mono numbers.

## 8. DARK MODE PROTOCOL
Dual-mode default. Token strategy: Tailwind dark: variant OR CSS semantic vars — pick one. Don't prescribe colors (brief decides); enforce contrast, hierarchy parity, brand fidelity, no pure #000/#fff. Respect prefers-color-scheme; add toggle if either mode loses brand. Test both modes before finishing.

## 9. AI TELLS (Forbidden)
9.A no neon/outer glows, no pure black, no oversaturated accents, no gradient text on big headers, no custom cursors. 9.B avoid Inter default, no oversized scream H1s, serif only editorial/luxury. 9.C mathematically clean spacing, NO 3-equal-column feature cards. 9.D no Jane Doe/John Doe names, no egg avatars, no fake-perfect numbers, no Acme/Nexus/SmartFlow, no filler verbs (Elevate/Seamless/Unleash/Next-Gen/Revolutionize). 9.E no hand-rolled SVG icons, no div fake screenshots, no broken Unsplash (use picsum seed), shadcn never default state.
9.F Production tells banned: version labels in hero (V0.6/BETA/EARLY ACCESS); "Brand · No.01" sub-eyebrows; section-number eyebrows (00/INDEX, 001·Capabilities, 06·how it works); "01/4" pagination; "Scroll·001" cues; range-label eyebrows; middle-dot rationed (max 1/line); decorative colored status dots; <br>-broken italic headlines; vertical rotated text; crosshair/hairline decoration; div fake product UI in hero; fake version footers; "Quietly in use at"; "Field notes/On our desks" poetic labels; mock-humble references; weather/locale strips; micro-meta sentences under eyebrows; generic step labels (Stage/Step/Phase 1/2/3); pills/labels overlaid on images; photo-credit captions as decoration; version footers on marketing pages; live-stock counters; hero-bottom decoration strips (BRAND.MOTION.SPATIAL.); floating top-right sub-text; scoring bars with filled tracks; scroll cues.
9.G EM-DASH BAN — completely banned everywhere visible (—  and – as separator). Only regular hyphen `-` and math minus allowed. A single em-dash fails pre-flight. Non-negotiable.

## 10. REFERENCE VOCABULARY
Hero: Asymmetric Split, Editorial Manifesto, Video/Media Mask, Kinetic-Type, Curtain-Reveal, Scroll-Pinned. Nav: Dock Magnification, Magnetic Button, Gooey Menu, Dynamic Island, Radial Menu, Speed Dial, Mega Menu. Layout: Bento, Masonry, Chroma Grid, Split-Screen Scroll, Sticky-Stack. Cards: Parallax Tilt, Spotlight Border, Glass Panel, Holographic Foil, Swipe Stack, Morphing Modal. Scroll: Sticky Stack, Horizontal Hijack, Sequence Scroll, Zoom Parallax, Progress Path, Liquid Swipe. Galleries: Dome, Coverflow, Drag-to-Pan, Accordion Slider, Hover Trail, Glitch. Type: Kinetic Marquee, Text Mask Reveal, Scramble, Circular Path, Gradient Stroke, Kinetic Grid. Micro: Particle Button, Pull-to-Refresh, Skeleton Shimmer, Directional Hover Fill, Ripple, SVG Line Draw, Mesh Gradient, Lens Blur. Libraries: Motion default; GSAP+ScrollTrigger for scrolltelling (isolated leaf + cleanup); Three.js for canvas/3D (isolated); never mix GSAP/Three with Motion in one tree.

## 11. REDESIGN PROTOCOL
11.A Detect mode: greenfield / redesign-preserve / redesign-overhaul; ask once if ambiguous. 11.B Audit first: brand tokens, IA, content blocks, patterns to preserve/retire, existing dial reading (that's the start point not baseline), SEO baseline (migration = #1 risk). 11.C Preserve: don't change IA/slugs/anchor IDs/nav labels; extract brand color before 4.2 (purple stays purple); preserve copy voice; honor a11y wins; respect analytics events. 11.D Levers in order: typography → spacing/rhythm → color recalibration → motion layer → hero recomposition → full block replace (last). 11.E Targeted evolution when IA/content/SEO sound; full redesign when structural debt; greenfield when brand changes. 11.F Never change silently: URL/slugs, nav labels, form field names/order, logo, legal/consent copy.

## 14. FINAL PRE-FLIGHT CHECK (run every box before shipping; not optional)
Design read declared; dials reasoned; design system chosen or aesthetic labeled; redesign mode + audit if applicable; ZERO em-dashes anywhere; theme lock; color-consistency lock; shape-consistency lock; button contrast; no CTA wrap; form contrast; serif discipline (not Fraunces/Instrument, brand-justified, different from last project); premium-consumer palette not beige+brass; italic descender clearance; hero fits viewport; hero pt-24 max; hero ≤4 text elements; eyebrow count ≤ ceil(sections/3); no split-header; zigzag ≤2 consecutive; no duplicate CTA intent; logo wall = logos only, under hero, real SVG marks; bento background diversity; copy self-audit; motion motivated; marquee ≤1; nav one line ≤80px; ≥4 layout families / 8 sections; bento exact cell count; long lists use real components; real images (no div screenshots, no hand-rolled decorative SVG, no pure-text minimalism); no pills/captions overlaid on images; no version footers; no micro-meta sentences; no hero-bottom decoration strip; no floating corner sub-text; no filled-track score bars; no locale/time/weather strips; no scroll cues; no version labels in hero; no section-number eyebrows; no decorative dots; no border-t+border-b on every row; sane content density; quotes ≤3 lines; motion shown if MOTION>4; GSAP canonical skeletons; no addEventListener scroll; reduced motion for MOTION>3; dark mode tested both; mobile collapse explicit; min-h-[100dvh]; useEffect cleanup; empty/loading/error states; cards omitted where possible; icons from allowed library; motion isolated in client leaves; no Section-9 tells; CWV plausible; one design system per project.
If a single box can't be honestly ticked, the page is not done.

## 13. OUT OF SCOPE
Not for dashboards/dense product UI (use Fluent/Carbon/Atlassian/Polaris), data tables (TanStack/AG Grid), multi-step wizards, code editors (Monaco/CodeMirror), native mobile (HIG/Material), realtime collab UIs. If the brief is one of these, say so, point to the right tool, apply only the marketing/landing parts that fit.
